WATCHMAN PERMISSION ERROR - FIX DOCUMENTATION
==============================================

ISSUE
-----
Error: std::__1::system_error: open: /Users/divakarchoudhary/Downloads/enout-main/mobile: Operation not permitted

ROOT CAUSE
----------
macOS is blocking Watchman from accessing the project directory due to Full Disk Access permissions.

SOLUTION OPTIONS
----------------

Option 1: Grant Full Disk Access (Recommended)
1. Open System Settings > Privacy & Security > Full Disk Access
2. Click the "+" button
3. Navigate to and add:
   - /usr/local/bin/watchman (if installed via Homebrew)
   - /opt/homebrew/bin/watchman (if installed via Homebrew on Apple Silicon)
   - /usr/local/Cellar/watchman/*/bin/watchman (Homebrew location)
4. Restart Terminal and try again

Option 2: Use Start Script (Quick Fix)
Run: ./mobile/start-android.sh
This script disables Watchman and starts Expo without it.

Option 3: Environment Variable
export CI=true
export WATCHMAN_DISABLE=1
npx expo start --android

Option 4: Disable Watchman Service
watchman shutdown-server
Then ensure CI=true is set before starting Expo

----------------
Updated metro.config.js to limit watchFolders to mobile directory only.
This reduces the scope of file watching and may avoid permission issues.

WATCHMANCONFIG FIX
------------------
